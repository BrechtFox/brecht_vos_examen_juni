# Opdracht – Dynamische webapp met JavaScript

## Doel van de opdracht

Maak een **eigen, dynamische webapplicatie** met minstens twee pagina’s.  
De app moet gebruikers toelaten om gegevens te beheren: toevoegen, bekijken, aanpassen, verwijderen, sorteren, filteren en samenvatten.  

Alle ingevoerde gegevens moeten bewaard blijven, ook als iemand de browser sluit en later terugkomt (gebruik daarvoor localStorage).

Kies zelf een interessant en origineel onderwerp. Enkele voorbeelden ter inspiratie:

- Mijn game collectie
- Favoriete films / series met beoordeling
- Persoonlijke boekenlijst met leesstatus
- Dagelijkse taken of projecten
- Uitgaven / uitgavenbeheer
- Eenvoudige fitness- of sportlog
- Receptenverzameling
- Eigen flashcard-systeem voor studeren
- Verzamelobjecten (sneakers, vinyl, planten…)

**Niet toegestaan:** een te nauwe kopie maken van de voorbeeld citaten-app. Gebruik die wel als technische inspiratie, maar maak iets dat duidelijk je eigen idee en aanpak is.

## Regels rond originaliteit

Dit is **je persoonlijk werk**.  
Code van medestudenten, GitHub-repositories of kant-en-klare oplossingen overnemen en als je eigen project indienen = plagiaat.  

Je mag wél tutorials volgen, documentatie raadplegen, Stack Overflow gebruiken of zelfs AI-tools inschakelen. Vermeld in dat geval altijd duidelijk je bronnen (link, gebruikte prompt, artikel…). Je moet de code die je gebruikt of aanpast wél volledig begrijpen en kunnen uitleggen tijdens het mondeling examen.

## Technische minimumeisen

- Werk in de GitHub Classroom repository
- Deploy de app via **GitHub Pages**
- Commit regelmatig met duidelijke, korte berichten
- Maak een overzichtelijke **README.md** (gebruik Markdown netjes: kopjes, lijsten, vetgedrukt, codeblokken…)
- Minstens **twee pagina’s**:
  - één overzicht / lijstweergave
  - één pagina om items toe te voegen (en idealiter ook te bewerken)
- Aantrekkelijke, **responsieve** HTML + CSS (denk aan CRAP: Contrast, Repetition, Alignment, Proximity)
- Gebruik **localStorage** om alles lokaal op te slaan
- Pas de **DOM actief aan** (items toevoegen, verwijderen, updaten)
- Voeg idealiter een **bewerk-functie** toe (potloodje / edit-knop)
- Schrijf voldoende **functies** en wissel af tussen klassieke `function` en arrow functions `() =>`
- Werk met **arrays van objecten**
- Pas verschillende array-methodes toe (push, splice, sort, filter, map, reduce…)
- Kies zelf je manier van lussen (klassieke for, forEach, for…of, higher-order functies…)
- Gebruik **const** standaard – alleen **let** wanneer je een waarde echt moet wijzigen
- Koppel minstens **drie verschillende events** met addEventListener (klik, input, submit…)
- Voer **eenvoudige berekeningen** uit op numerieke waarden (totaal, gemiddelde, hoogste/laagste waarde, aantal items…)
- Pas het **DRY-principe** toe: herhalende code → functie van maken
- Splits je JavaScript op in **meerdere bestanden** (gebruik ES modules)

## Hoe wordt dit beoordeeld?

- Staat de app **foutloos** en doet alles wat je belooft?
- Is de **gebruikservaring logisch** en intuïtief?
- Toon je **eigen inzicht en creativiteit**? (geen minimale kloon van de les-app)
- Heb je technieken gebruikt die verder gaan dan wat in de les aan bod kwam? → extra punten
- Je mag op het mondeling examen extra toelichting geven over slimme keuzes of moeilijkere stukken
